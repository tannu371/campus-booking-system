Problem & Scope

The Problem

  • Manual room booking leads to double-bookings and conflicts.
  • No visibility for admins on who booked what and when.
  • No governance — controlled rooms need approval workflows.
  • No trail of booking actions for audits.

Our Solution

  • Self-service booking for students, staff & faculty.
  • Real-time availability with an interactive calendar.
  • Approval workflow for controlled rooms.
  • Audit logs with 19 action types for full traceability.
  • Atomic conflict detection via MongoDB transactions.

Architecture Layers

  • Routes → request validation (400 VALIDATION_ERROR)
  • Middleware → JWT auth + admin guards + rate limiting
  • Controllers → business rules, conflict checks
  • Models → Mongoose schemas & unique indexes
  • Utils → validators, audit logger, email templates

Deployment-Ready

  • Safe for PM2 cluster / Kubernetes replicas
  • DB-level enforcement prevents double-booking
  • CORS allowlist, rate-limiting, pinned dependencies
  • Environment-driven config (JWT TTLs, SMTP, seed)

User Features

Discover & Browse

  • Filter by type, building, capacity
  • Interactive calendar showing real-time availability
  • Room detail view with schedule visualization

Book & Manage

  • Create booking with client + server-side validation
  • Unique confirmation code generated per booking
  • Role-based limits: admin 50 · faculty 20 · staff 10 · student 5
  • Cancellation with reason tracking

Check-In System

  • Check-in only allowed during booked time window
  • Timezone-stable via UTC datetimes
  • Marks no-shows automatically for analytics

Notifications

  • Email confirmations via Nodemailer
  • Approval / rejection notifications
  • HTML-escaped email templates (XSS safe)

Admin Capabilities

Dashboard & Analytics

  • Live stats: active bookings, pending approvals, users
  • Booking trends & no-show rates
  • Room utilization reports

Room Management

  • Add / edit / soft-delete rooms
  • Control which rooms require approval
  • Set capacity, type, building metadata

Booking Oversight

  • Approve / reject pending bookings
  • Override or cancel conflicting bookings
  • Alternative room suggestions on conflicts

User & Audit Controls

  • Change roles, suspend / activate users
  • 19-action audit log with pagination & filtering
  • Recent-activity feed + action-type breakdown

Security Hardening

Authentication

  • Short-lived JWT access tokens (1h)
  • Rotating refresh tokens (7d)
  • Token revocation via jti tracking
  • Active-status check blocks suspended users

Input & Injection

  • NoSQL injection mitigation (primitive type checks)
  • Escaped regex inputs to prevent ReDoS
  • Schema validation at route layer
  • HTML escaping in email output

Access Control

  • Role-based guards on every admin endpoint
  • Role-based booking quotas
  • Soft-delete visibility enforcement

Infrastructure

  • CORS allowlist (trusted origins only)
  • Rate limiting on auth & admin routes
  • Pinned runtime dependencies
  • CSP headers + hardened error responses

cd backend
npm test                  # all 129 tests
npm run test:coverage     # coverage report
npm run test:security     # security-focused

Walkthrough

  1. Browse & Filter rooms by capacity / building.
  2. Create a booking as the student — show confirmation code.
  3. Trigger a conflict — book same slot, observe rejection + alternatives.
  4. Approval flow — request a controlled room, approve from admin tab.
  5. Check-in window — try before window (fails), during (succeeds).
  6. Admin ops — soft-delete a room; it disappears from user view.
  7. Audit log — scroll the activity feed showing every action.
